# Not Yet List

Use this file to protect the project from bloat. These ideas are not rejected forever, but they should not be added during the pilot unless a later decision changes that.

## Not during the pilot

- Separate blog page.
- Public polls.
- Comments system.
- Mailing list.
- Complex CMS.
- Complex build system.
- React or heavy JavaScript framework unless genuinely necessary.
- Mass-emailing public officials.
- Mass mobilisation.
- Public accusations without strong evidence.
- Legal threats.
- Anything that could look like harassment, dogpiling or spam.
- Over-designed campaign panels or fake campaign gloss.
- Silicon Valley AI hype.
- Party-political branding or colour coding.

## Why these are paused

The pilot needs to prove the basic method first:

1. Pick one issue.
2. Read the public sources.
3. Check the existing route.
4. Make one reasonable ask.
5. Log what happened.
6. Correct mistakes.
7. Keep going calmly.

Anything that distracts from that rhythm should wait.

## Review point

Review this list after Week 4 during the pilot review.
